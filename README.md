### Deep in Lost City

> A solo, end-to-end ownership exercise across engineering and product, deliberately run under fixed time, scope, and budget (the Iron Triangle). Applied POCO architecture and the Single Responsibility Principle to keep the code clean under deadline pressure, solved critical WebGL issues (scene freezing, audio leaks, costly Gaussian Blur), and tracked every major call in a living Decision Log to enable conscious pivots.

### Game Market Intelligence

> A data aggregation tool for Game Producers, built to fill a gap felt firsthand in game jams: no single trustworthy source for market data. Every architecture decision is documented as a formal ADR, backed by real proofs of concept. The database choice, for instance, was settled by testing Neon, Azure SQL Free, Supabase, and Aiven under real conditions: Azure SQL was rejected after a critical cold-start failure, while Neon handled the same test with no intervention. The delivery pipeline runs fast checks on develop and full integration testing (via Testcontainers) on Pull Requests to main, deploying automatically only after all checks pass.

### Classroom Automation

> I worked as an instructor in two distinct courses: Game Development (Unity and C#) and Front-end Web Programming (HTML, CSS, and introductory JavaScript). During this teaching period, I identified an opportunity to optimize lab setup time. Routine permission configuration (Execution Policies) and software preparation consumed valuable class time. Because of this, I developed a PowerShell automation pipeline. The solution reduced environment setup to 2 minutes via a 'one-click' process, drastically optimizing class throughput and my own productivity.
